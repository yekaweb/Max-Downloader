# Roadmap: طراحی و ساخت رابط کاربری پنل مدیریت وب (Neumorphism UI)

## 🎯 هدف
در حال حاضر پنل مدیریت وب بات (`dlbot_web_panel`) فقط به عنوان یک واسط برنامه‌نویسی (API Server) و با امنیت JWT عمل می‌کند و رابط کاربری (UI) گرافیکی برای ورود (Login) و نمایش داده‌ها ندارد. هدف از این رودمپ، طراحی و اتصال یک صفحه لاگین گرافیکی امن و داشبوردی کاملاً مدرن بر پایه طراحی **Neumorphism (نئومورفیسم)** است تا مدیریت ربات به ساده‌ترین و زیباترین شکل ممکن انجام شود.

## 🎨 استایل و معماری طراحی (Neumorphism Theme)
نئومورفیسم یک سبک طراحی مینیمال و نرم است که با استفاده از سایه‌های ملایم روشن و تاریک، حالتی برجسته (کمی شبیه به پلاستیک فشرده یا متریال‌های مات) ایجاد می‌کند. در این طراحی از:
- رنگ‌بندی یکپارچه ملایم (مانند خاکستری روشن، آبی مات، یا دارک‌مود عمیق)
- دکمه‌هایی که به جای حاشیه، با سایه عمق پیدا می‌کنند
- طراحی واکنش‌گرا (Responsive) برای موبایل و دسکتاپ
استفاده خواهیم کرد.

## 📅 فازهای توسعه و پیاده‌سازی (Implementation Phases)

### 🔴 فاز ۱: ایجاد سیستم احراز هویت مبتنی بر کوکی (Cookie-based Auth)
در فریم‌ورک FastAPI، برای لاگین در مرورگر (علاوه بر API)، نیاز داریم که توکن در Cookie ذخیره شود تا خطای `Not authenticated` در لود صفحات HTML رفع شود.
* [ ] آپدیت فایل `web/auth.py` برای خواندن JWT 토کن از دو مسیر: هدر `Authorization` (برای API) و کلید `access_token` از کوکی‌ها (برای مرورگر).
* [ ] ایجاد روتر جدید `login` که یوزرنیم و پسورد ادمین را بگیرد و `access_token` را در قالب کوکی امن (`HttpOnly`) در مرورگر تنظیم کند.
* [ ] ساخت مسیر `/logout` برای پاک کردن کوکی و خروج از حساب کاربری.

### 🟡 فاز ۲: طراحی قالب گرافیکی لاگین (Neumorphic Login Page)
* [ ] ایجاد فایل `web/templates/login.html`.
* [ ] پیاده‌سازی استایل‌های پایه CSS نئومورفیسم (رنگ پس‌زمینه، کادر ورود برجسته، و دکمه‌های نرم با سایه داخلی هنگام کلیک).
* [ ] متصل کردن فرم لاگین به سیستم احراز هویت فاز ۱ (ارسال درخواست به بک‌اند و ریدایرکت خودکار به داشبورد در صورت موفقیت).
* [ ] نمایش پیام‌های خطای گرافیکی (مثل "رمز عبور اشتباه است") در صورت عدم موفقیت.

### 🟢 فاز ۳: طراحی داشبورد مدیریتی (Neumorphic Dashboard UI)
* [ ] اصلاح فایل `web/templates/dashboard.html` بر اساس استایل Neumorphism.
* [ ] طراحی کارت‌های اطلاعات (Cards) برای نمایش آمار (مانند "تعداد کاربران"، "دانلودهای امروز" و "درآمد/سکه").
* [ ] ساخت منوی ناوبری (Sidebar/Navbar) برجسته برای جابجایی بین بخش‌های مختلف (کاربران، گزارشات، تنظیمات).
* [ ] پیاده‌سازی نمودارها (Charts.js یا ApexCharts) درون کارت‌های دارای عمق (Inset Shadows).

### 🔵 فاز ۴: اتصال نهایی بک‌اند و فرانت‌اند (Integration & Polishing)
* [ ] جایگزینی داده‌های Mock (تستی) در فایل `web/app.py` با داده‌های واقعی دیتابیس بات (گرفتن تعداد واقعی یوزرها از `users` router).
* [ ] ایجاد صفحات فرعی مانند `web/templates/users.html` برای نمایش لیست کاربران و اعمال قابلیت بن/آن‌بن کردن مستقیماً از وب‌پنل.
* [ ] تست نهایی ورود، ماندگاری نشست (Session Persistence) روی مرورگر و امنیت پنل.
