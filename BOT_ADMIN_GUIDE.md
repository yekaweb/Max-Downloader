# ⚙️ راهنمای مدیریت DLBot - برای ادمین‌ها

---

## 👤 دسترسی ادمین

### چه کسی می‌تواند ادمین باشد؟

```
از فایل .env:
ADMIN_IDS=535435383,242397701

شما می‌توانید یک یا چند نفر را ادمین کنید
(با کاما جدا کنید)
```

---

## 🔑 دستورات ادمین

| دستور | توضیح | مثال |
|-------|--------|------|
| `/admin` | ورود به پنل | `/admin` |
| `/admin stats` | مشاهده آمار | `/admin stats` |
| `/admin broadcast` | ارسال پیام عمومی | `/admin broadcast` |
| `/admin users` | مدیریت کاربران | `/admin users` |
| `/admin ban <user_id>` | مسدود کردن کاربر | `/admin ban 123456` |
| `/admin unban <user_id>` | رفع مسدودیت | `/admin unban 123456` |

## 📢 ارسال پیام عمومی (Broadcast)

### مثال: Broadcast ساده

```
دستور:
/admin broadcast

سپس پیامی بنویسید:
سلام! ربات بهبود یافت! 🎉
```

**ربات به تمام کاربران ارسال می‌کند:**
```
سلام! ربات بهبود یافت! 🎉
```

---

### مثال: Broadcast با Inline Buttons

```
پیام:
🎁 دانلود رایگان!

Buttons:
- [📥 دانلود] → /download
- [📚 راهنما] → /help
```

---

### بهترین تمرین‌ها

```
✅ DO:
- پیام‌های کوتاه و مفید
- استفاده از emoji
- ارسال در ساعات کاری
- تست پیش از ارسال

❌ DON'T:
- Broadcast بیش از 3 بار روزانه
- پیام‌های تبلیغاتی بیش از حد
- ارسال در ساعات شب
- متن‌های بیش از 500 حرف
```

---

### مسدود کردن کاربر

```
دستور:
/admin ban 123456

نتیجه:
✅ کاربر #123456 مسدود شد
⚠️ دیگر نمی‌تواند دانلود کند
```

### رفع مسدودیت

```
دستور:
/admin unban 123456

نتیجه:
✅ کاربر #123456 مجاز شد
✨ دوباره می‌تواند استفاده کند
```

## ⚠️ مدیریت خطاها

### مشکلات متداول

#### مشکل: کاربران زیاد غیرفعال

```
❌ مسئله:
1000 کاربر فعال نیستند

✅ حل:
1. /admin broadcast ارسال کنید
2. پیام جذاب بنویسید
3. یک دکمه دانلود اضافه کنید
```

---

#### مشکل: بسیاری از خطاها در لاگ

```
❌ مسئله:
ERROR در logs بسیار زیاد است

✅ حل:
1. لاگ را بررسی کنید
2. مشکل را شناسایی کنید
3. Server را Restart کنید
```

---

#### مشکل: Disk Space کم

```
❌ مسئله:
"درایو پر است!"

✅ حل:
1. فایل‌های کش قدیمی را حذف کنید
2. لاگ‌های قدیمی را پاک کنید
3. Database را Optimize کنید
```

## 📈 بهینه‌سازی

### افزایش سرعت

```
1. Database Optimization:
   - Indexes را بررسی کنید
   - Duplicate records را حذف کنید

2. Cache بهتر:
   - Redis را Restart کنید
   - TTL را بررسی کنید

3. Server Resources:
   - CPU/RAM را نظارت کنید
   - Process‌های غیرضروری را بسته کنید
```

---

### کاهش استفاده Storage

```
1. Cache را تمیز کنید:
   - فایل‌های > 48 ساعت قدیمی
   - دانلودهای تکراری

2. Logs را Rotate کنید:
   - لاگ‌های > 30 روز قدیمی
   - اندازه هر فایل < 100 MB

3. Database را Cleanup کنید:
   - رکوردهای حذف‌شده
   - Transactions قدیمی
```
